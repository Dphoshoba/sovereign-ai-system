\# PURPOSE



This document defines the daily operating procedure for Sovereign.



The objective is simple:



Keep the business healthy, growing, learning, publishing and executing.



Sovereign is designed to operate as an executive operating system.



Every day follows the same cycle:



Observe → Learn → Forecast → Decide → Approve → Execute → Measure



\---



\# DAILY EXECUTIVE CHECKLIST



Estimated Time: 15–30 Minutes



\---



\# STEP 1 — OPEN MISSION CONTROL



Navigate to:



```text

Admin → Mission Control

```



Review:



\* Overall Health

\* Executive Status

\* Finance Score

\* Operations Score

\* Strategy Score



Immediate Questions:



```text

Is health improving?

Is health declining?

Are any scores below 40?

```



If Overall Health < 40:



```text

Activate Recovery Mode

```



\---



\# STEP 2 — REVIEW EXECUTIVE ALERTS



Navigate to:



```text

Admin → Executive Alerts

```



Review:



\* Critical Alerts

\* High Priority Alerts

\* New Risks

\* Escalations



Required Action:



```text

Clear critical alerts first.

```



Rule:



Never ignore critical alerts for more than 24 hours.



\---



\# STEP 3 — REVIEW EXECUTIVE AUTOPILOT



Navigate to:



```text

Admin → Executive Autopilot

```



Review:



\* Recommended Actions

\* Recovery Actions

\* Priority Actions



Questions:



```text

What is Sovereign recommending today?

Which actions are blocking progress?

```



\---



\# STEP 4 — REVIEW APPROVAL QUEUE



Navigate to:



```text

Admin → Governance

```



Review:



\* Pending Approvals

\* Executive Reviews
