# EV-KOS Enterprise Capability Matrix

| Area | Capability | Alpha Position | Beta Requirement |
| --- | --- | --- | --- |
| Foundation | Tenant and governance foundation | Contract complete | Select and enforce auth provider |
| Topology | Organizations, workspaces, departments, teams | Contract complete | Map persistence and memberships |
| Guards | Tenant, workspace, and organization boundaries | Preview-ready | Enforce guards on routes |
| Audit | Evidence packets and trust model | Preview-ready | Persist immutable audit evidence |
| Approval | Review boards and decision packets | Preview-ready | Execute governed review workflows |
| Policy | Versioning, exceptions, drift, change control | Preview-ready | Enforce policy lifecycle |
| Deployment | Gates, promotion, release tiers | Contract complete | Wire into CI and hosting |
| Execution | Enterprise actions and mutations | Blocked by design | Requires Beta authorization stack |

The capability matrix intentionally separates architectural readiness from
runtime enablement.
