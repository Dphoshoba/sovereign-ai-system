# EV-KOS Enterprise Compliance

## Purpose

EA-4 defines a compliance surface for planning.

## Control Families

- Access.
- Tenant isolation.
- Audit.
- Data boundary.
- Publishing.
- AI safety.

## Readiness

The compliance surface is defined but not enforced. Publishing, AI generation, database writes, graph writes, sessions, JWT, and auth integration remain blocked.

## Required Before Enforcement

- Authentication provider approval.
- Route authorization enforcement.
- Audit persistence plan.
- Telemetry redaction plan.
- Rate limiting.
- Review-board operating model.
